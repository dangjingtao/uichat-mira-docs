认识 Mira · 产品总览

从聊天出发，最终回到你身边。

UIChat Mira 是一个 local-first desktop workspace：聊天只是入口，模型、角色、知识、MCP、工具与微应用在同一个本地工作空间里协同。

Mira 不是把所有能力堆进一个输入框，而是给长期的人机协作建立一个有边界、有记忆、有工具、也有退路的空间。

产品地图

Mira 当前可以按五个产品域理解：对话工作区、模型与 Provider、知识与评测、角色与微应用，以及工具与集成。

对话工作区

线程保存任务上下文，消息记录人和模型的交流，Agent run 保存一次推理与工具执行的过程。附件、知识检索、上下文摘要和运行状态共同进入对话。

知识与评测

知识系统负责让模型接触用户自己的材料，评测系统负责回答“这次改动真的更好吗”。Markdown 工作区保留来源，适合维护长期判断和项目背景。

角色、工具与微应用

Role 定义任务视角和行为约束，工具通过 Harness 注册和暴露，微应用复用共享运行时承载不同工作方式。

本地优先，不是本地孤岛

对话、配置、知识与工作状态应当首先属于用户，而不是某个远端服务的附属物。Provider、MCP 和集成可以连接外部能力，本地负责保存状态与边界。

可控的自主

Planner 可以决定检索、调用工具、生成答案或继续推理，但执行权始终受 Harness、Policy 与 Approval 约束。决策和执行必须分开。

证据先于答案

“证据足以回答”不等于“任务已经完成”。Retrieve 与 ToolNode 只写入客观证据，再回到 Planner，由 Planner 根据新证据决定继续、生成或停止。

作者与项目

UIChat Mira 由 Tomz Dang 发起并持续构建。项目版本为 0.7.1，源码与文档公开在 GitHub。

桌面运行时

Renderer 使用 React 与 Vite；Electron 负责窗口、生命周期和 preload 边界，Tauri 作为并行壳保留；Fastify 与 SQLite 承载本地后端和持久化。

Agent V1.5 运行回路

运行从 prepareContext 进入 nextActionPlanner。Planner 可以选择 Retrieve、Tool、Generate 或错误终止，证据随后回到 Planner，避免额外决策节点猜测用户意图。

Harness 与工具边界

CapabilityMatch 回答系统能否提供能力，ToolExposure 回答模型此刻能否看到，Invocation 才是带具体 toolId 的执行请求。

Provider 与上下文

Provider 层负责连接模型，上下文层负责决定模型此刻真正能看到什么。能力来自配置和探测结果，不能只根据供应商名称猜测。

快速开始

如果你准备运行产品，可以继续阅读下面的安装、Provider 配置和第一次对话示例。

  1. 从 GitHub Releases 下载对应系统的安装包。
  2. 首次启动时创建本地配置，并选择一个模型 Provider。
  3. 在工作区创建线程，开始第一条对话。
  1. 从 GitHub Releases 下载对应系统的安装包（macOS / Windows / Linux）。
  2. 首次启动时，Mira 会在 ~/.mira/config.yaml 创建一份默认配置文件。
  3. （可选）如果只想用命令行，可以通过包管理器安装 CLI 版本：
# macOS / Linux
curl -fsSL https://get.uichat-mira.dev | sh

# 安装完成后验证版本
mira --version
Windows 用户请使用安装包中的 PowerShell 脚本，命令行参数与 macOS/Linux 版本保持一致。

配置第一个模型

打开配置文件，填入至少一个模型提供方。云端模型需要 API Key，本地模型只需要指向本地服务地址：

providers:
  - name: claude
    type: anthropic
    api_key_env: ANTHROPIC_API_KEY   # 从环境变量读取，不写进配置文件
  - name: local-qwen
    type: ollama
    base_url: http://127.0.0.1:11434
    model: qwen2.5:14b
字段 说明 是否必填
name 模型提供方的唯一标识，用于对话中切换 是
type 提供方类型：anthropic / openai / ollama / custom 是
api_key_env 存放密钥的环境变量名（云端模型需要） 视类型而定
base_url 本地或私有部署的服务地址 本地模型需要

第一次对话

启动 UIChat Mira 后，在顶部模型切换栏选择一个已配置好的提供方，就可以直接开始对话：

  1. 点击左上角「新建对话」。
  2. 在模型切换栏选择 claude 或 local-qwen。
  3. 输入消息并发送——本地模型的回复不会经过任何外部服务器。
如果本地模型没有响应，先确认 Ollama 服务是否已经启动（ollama serve），并检查 base_url 端口是否一致。
