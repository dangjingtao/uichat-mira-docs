# MCP 服务边界怎么划，才不会变成一锅粥

服务边界如果含糊，最后每个工具都像是“顺手加进去的”。

## 我们采用的原则

- 以用户意图分边界，不只看技术协议
- 优先稳定输入输出，而不是内部实现
- 工具发现和工具执行分开看待

## 真正帮到我们的经验

工具名越像内部实现，未来越难迁移。用户要的是能力，不是组件树。
