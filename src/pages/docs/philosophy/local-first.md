# 本地优先，不是本地孤岛

Local-first 是 Mira 的产品起点：用户的对话、配置、知识与工作状态应当首先属于用户，而不是某个远端服务的附属物。

## 为什么从本地开始

源码把持久化建立在 SQLite 上，桌面端同时维护 Electron 主路径与 Tauri 并行壳。模型 Provider、工具和集成可以连接外部世界，但工作舱本身不因某一家云服务变化而失去基础形态。

## 本地不意味着封闭

Mira 通过统一的 Provider 接口、MCP、Integration 与 Harness 接入模型和工具。本地负责保存状态与边界，外部能力负责扩展可能性；两者不是互斥关系。

## 用户拥有最终控制权

本地优先真正保护的是“可退出性”：配置可见、数据可迁移、调用可审计。即使 Agent 能够连续行动，授权、风险和结果仍应回到用户可理解的界面。

## 工程上的代价

这种选择要求处理桌面打包、数据库迁移、跨进程通信、平台差异与离线状态。Mira 接受这些复杂度，因为控制权不是一句口号，而要落实到存储位置和运行边界。
