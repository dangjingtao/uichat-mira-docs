# 企业集成

企业集成页负责把 Mira 接到企业协作平台，并将外部消息入口、主动通知和内部微应用组织成可管理的接入关系。

![企业集成概览](/images/guide/enterprise-overview.svg)

*当前版本已经接入企业微信，并为飞书、钉钉保留了后续平台入口。*

## 平台与实例

企业集成不是一个全局开关。

页面先区分平台，再管理某个平台下的接入实例。一个实例可以包含多种接入方式，并在完成配置后参与聊天、自动化流程和机器人回复链路。

当前截图中可以看到：

- 企业微信已经可用；
- 飞书与钉钉仍处于规划状态；
- 当前实例会显示名称、状态、主要方式和已接入数量；
- “配置接入”用于继续补齐平台级配置。

## 两种企业微信入口

### 智能机器人

智能机器人用于接收群聊或私聊消息，把问题送入 Mira 的处理链路，并把回答返回企业微信。

它可以绑定微应用。例如截图中的智能机器人已经关联 `Default Knowledge Query`，因此能够复用本地知识库问答能力。

### Webhook 机器人

Webhook 机器人更适合主动通知：由聊天、任务或自动化流程向企业微信群推送消息。

它与智能机器人承担的方向不同：

- 智能机器人偏向“外部消息进入 Mira，再返回回答”；
- Webhook 机器人偏向“Mira 主动把消息推送出去”。

## 为什么接入与微应用分开

企业平台负责认证、连接和入口协议；微应用负责具体业务逻辑。

这层分离可以避免把“企业微信机器人”直接写死成某一种知识库问答，也允许同一个企业入口在未来绑定不同能力。

可以把关系理解为：

```text
企业平台
→ 接入实例
→ 接入方式 / Access Point
→ 微应用
→ 知识、工具或业务流程
```

## 当前边界

- 当前主要落地平台是企业微信；
- 智能机器人可以承接问答入口；
- Webhook 机器人用于主动推送；
- 飞书和钉钉仍属于后续方向；
- 配置完成不代表所有能力都会自动暴露，实际行为仍取决于绑定的微应用和运行时策略。
