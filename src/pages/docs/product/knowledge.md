# 知识与评测

知识系统负责让模型接触用户自己的材料，评测系统负责回答“这次改动真的更好吗”。

## 知识库

源码中的知识域覆盖文档、分块、向量索引与工作区。摄入流程把原始文件转换为可检索单元，并保留来源，使回答能够回到证据。

## Markdown Workspace

除了传统上传与索引，Mira 也把 Markdown 视为可直接维护的知识工作区。文本可读、可版本化，适合把人的长期判断和项目背景留在本地。

## 检索进入 Agent

Retrieve 节点产生的是客观 evidence。它不决定答案，只把相关材料交还 Planner；相关性不足时，Planner 仍需继续澄清或选择其他行动。

## Evaluation Workbench 与 Center

评测域分为 Workbench 和 Center：前者组织数据预检与运行，后者保存历史、指标和报告。评测不只是模型排行榜，而是让提示词、Provider、知识和工具变化可以复现地比较。
