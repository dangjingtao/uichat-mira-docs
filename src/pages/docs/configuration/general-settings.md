# 通用设置与个人数据

「通用」页负责管理当前账户的基础信息、界面偏好和少量应用级操作。

## 当前账户

页面顶部显示当前用户名和账户角色，并提供修改密码入口。

修改密码时需要输入当前密码，新密码至少 6 位，且不能与当前密码相同。保存成功后，新密码立即生效。

这里显示的是 Mira 本地账户，不等同于模型 Provider、GitHub、企业微信或其他外部平台账号。

## 界面语言与主题

当前界面支持简体中文和 English。

主题用于改变界面的色彩气质，不会改变模型行为、知识检索方式或角色提示词。暗黑模式则在当前主题基础上切换明暗外观。

当前内置主题包括：

- 暖陶米色；
- 铁墨紫灰；
- 档案墨绿；
- 岩板海雾。

主题和语言属于应用偏好，适合按用户保存。

## 清理历史对话

「一键清理」是高风险操作。

它会永久删除当前账户的全部对话及其消息，并清空服务端日志。确认前应先检查是否有仍需保留的聊天、附件或执行记录。

需要特别区分：

- **对话与消息**：会被删除；
- **服务端日志内容**：会被清空；
- **知识库与评测**：不应被当作聊天数据理解；
- **用户绑定的工作空间文件**：不会因为清理聊天而被修改。

界面最终显示的清理统计可能同时包含线程、消息、媒体文件和相关工作状态，以实际确认弹窗和完成提示为准。

## SOCKS5 代理

当前页面可以保存 Host、Port、用户名和密码等 SOCKS5 出站代理配置。

但当前版本的页面说明明确表示：这部分目前主要是配置占位，尚未接入实际请求链路。保存成功不代表模型、MCP 或其他网络请求已经自动经过该代理。

留空 Host 或 Port 表示不启用代理。

## 数据安全建议

- 不要在截图、日志或公开文档中暴露 API Key、密码和代理凭据；
- 清理数据前先确认知识库、工作空间和聊天数据的区别；
- 导出日志用于排错时，应先检查其中是否包含本地路径、模型地址或请求信息；
- 公共电脑上使用后，应退出账户并避免保留长期凭据；
- 外部 Provider 的隐私策略由对应服务商决定，Mira 的本地优先不能替代对第三方条款的判断。
