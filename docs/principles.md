# 产品原则

## 多供应商，而非单模型绑定

Provider / Model Gateway 隔离协议差异，让产品能力高于某一家模型接口。

## 稳定闭环，而非盲目自治

Planner 判断任务是否完成；Policy 与 Harness 约束执行边界。失败必须可分类、可恢复、可观察。

## 少而清晰的能力面

避免工具爆炸。面向模型暴露稳定语义，把实现复杂度留在 Harness 内部。

::: tip 产品判断
能力数量不是核心指标。用户能否稳定完成任务，才是。
:::

## 可观察，而非黑盒魔法

记录请求、证据、耗时、Token 与成本，让失败能够定位，让判断能够复核。
