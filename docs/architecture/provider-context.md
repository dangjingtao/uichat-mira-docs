# Provider 与上下文

模型不是 Mira 的唯一中心。Provider 层负责连接模型，上下文层负责决定模型此刻真正能看到什么。

## 统一 Provider 契约

后端以 OpenAI-compatible 形态作为重要基线，同时为 LM Studio、Ollama、Cloudflare、火山等环境保留适配空间。统一接口覆盖连接测试、模型发现、生成与能力画像。

## 能力不能靠猜

不同模型是否支持工具、视觉、结构化输出或特定参数，需要来自配置和探测结果。运行时应根据能力画像降级，而不是把供应商名称当成能力结论。

## 上下文的组成

一次运行的上下文可以包含系统与角色指令、线程消息、摘要、附件、知识证据、工具观察和当前任务状态。prepareContext 负责整理输入，Planner 决定下一步如何使用。

## 控制上下文膨胀

上下文不是越多越好。来源、时效、相关性与 token 成本都需要显式管理；摘要不能覆盖关键证据，工具结果也不应无边界地重复注入。
