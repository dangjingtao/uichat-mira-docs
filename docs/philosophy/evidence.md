# 证据先于答案

对话系统最危险的时刻，往往不是它说“不知道”，而是它把推测包装成已经完成的事实。

## 两种完成不能混淆

**Evidence answerable** 表示当前证据足以回答问题；**Task completable** 表示外部任务确实可以继续执行并达到目标。检索到一段说明，不等于已经执行了操作；工具返回成功，也不等于业务目标一定成立。

## Evidence 回到 Planner

Mira 的 ToolNode 与 Retrieve 节点只写入客观证据，再回到 Planner。Planner 根据新证据决定继续、生成或停止，执行节点不替用户下最终结论。

## 失败也是证据

超时、权限不足、参数无效和结果为空，都应成为结构化观察，而不是被吞掉。一个可信工作舱必须允许用户看见哪里失败、为什么失败，以及下一步还能做什么。

## 不提前宣称成功

生成最终回答前，系统需要核对目标、调用结果和未满足条件。这个原则既服务于 Agent 运行时，也服务于文档：已实现、实验中与规划中的能力应被明确分开。
