# 下一段路

这一页描述从当前源码可以合理推导的工程方向，不把它们冒充已经交付的功能。

## 先巩固 V1.5

稳定 Agent 回路仍是第一优先级：补齐状态不变量、失败路径、节点级回归测试和端到端轨迹，让“删除 shadow decider”成为可持续的契约。

## 强化 Evidence

工具与检索结果需要更统一的 evidence 结构、来源标记和完成条件。系统应持续区分证据足以回答、动作已经执行和业务目标真正完成。

## 让评测进入日常开发

Evaluation Workbench 与 Center 可以进一步连接 Provider、知识、提示词和 Agent 变更，使每次优化都能在固定数据集和指标下复现。

## 扩展而不破坏边界

新增 Provider、MCP、集成和微应用时，继续复用 Harness、身份、权限、数据与审计能力，避免每个功能形成自己的隐式运行时。

## 完善用户交付

桌面安装、升级、备份、迁移、错误恢复与文档同步，是 local-first 产品走向稳定使用不可回避的一部分。
