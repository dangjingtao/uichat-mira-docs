# 对话工作区

Mira 的核心界面是一间持续存在的工作舱，而不是一次性问答框。

## 线程与消息

数据库模型包含 workspace、thread、message 与 agent run。线程保存任务上下文，消息记录人和模型的交流，Agent run 则保存一次推理与工具执行的过程。

## 角色与上下文

角色系统允许为不同工作方式设置指令与模型偏好。附件、知识检索、上下文摘要和运行状态共同进入对话，而不是被压扁成一段不可追踪的提示词。

## 可见的执行过程

Agent 在检索、规划、审批、调用工具和生成答案之间移动时，UI 可以呈现状态与证据。用户看到的不只是最终文本，也包括这段文本如何得到。

## 一个空间，多种入口

普通聊天、带知识的问答、微应用任务和集成消息最终都可以回到工作区。Mira 的目标不是堆叠孤立功能，而是让它们共享同一套身份、状态和审计语义。
